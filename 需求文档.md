# 广西地球信息产业发展研究网站 V1 需求文档

## 1. 项目目标

### 1.1 项目名称
广西地球信息产业发展研究

### 1.2 项目定位
本项目是一个面向公众传播的研究型门户网站，聚焦广西及相关区域的地球信息产业动态。网站围绕“企业动态、技术进展、政策观察”三条主线，对权威来源信息进行结构化整理，并以新闻摘要、词云、知识图谱等方式呈现。

### 1.3 建设目标
- 建立一个风格稳重、信息密度较高、适合研究机构门户定位的网站前台。
- 自动汇聚地球信息科学、遥感、测绘、时空信息、数字孪生等相关新闻。
- 以“可信来源白名单”为基础，确保新闻可追溯、可验证、可分类。
- 基于新闻数据生成关键词词云和广西专题知识图谱。
- 支持中文和英文界面切换，新闻内容保持原文语言展示。

## 2. 目标用户与使用场景

### 2.1 目标用户
- 公众用户：快速了解广西地球信息产业的政策、技术和企业动向。
- 研究人员：浏览结构化新闻线索，辅助研判行业趋势。
- 产业观察者：关注广西本地园区、企业、项目和产业应用机会。
- 项目参与方：用于展示产业研究成果、专题数据和可视化分析结果。

### 2.2 典型使用场景
- 用户从首页快速查看最新政策、企业和技术动态。
- 用户通过关键词检索新闻，例如“实景三维”“北斗应用”“低空经济”。
- 用户进入词云页，点击高频关键词后跳转到对应新闻筛选结果。
- 用户进入知识图谱页，查看广西相关政策、机构、园区、技术之间的关联关系。
- 用户在来源说明页确认新闻来源是否属于权威白名单。

## 3. 信息架构与页面规划

### 3.1 页面结构
网站首版采用前台站点 + 定时更新任务模式，不建设内容管理后台。

### 3.2 路由规划
- `/{lang}`：首页
- `/{lang}/news`：新闻列表页
- `/{lang}/news/{slug}`：新闻摘要详情页
- `/{lang}/word-cloud`：词云页
- `/{lang}/knowledge-graph`：广西专题知识图谱页
- `/{lang}/sources`：数据来源与抓取说明页
- `/{lang}/about`：项目介绍页

其中 `lang` 取值为 `zh` 或 `en`。

### 3.3 首页模块
首页包含以下模块：
- 项目主视觉与简介
- 最新新闻聚合区
- 企业动态、技术进展、政策观察三大栏目卡片
- 词云预览
- 广西知识图谱预览
- 白名单来源说明
- 项目方法与范围说明

### 3.4 新闻列表页
- 提供关键词搜索
- 提供栏目分类筛选
- 提供来源筛选
- 提供广西相关筛选
- 提供时间排序
- 列表项展示标题、摘要、来源、发布时间、关键词和跳转入口

### 3.5 新闻摘要详情页
详情页不做原文全文转载，仅展示：
- 标题
- 摘要/导语
- 发布时间
- 来源名称
- 原文链接
- 分类信息
- 关键词
- 地域标签
- 相关推荐

## 4. 新闻采集与可信规则

### 4.1 采集范围
首版重点覆盖以下领域的公开新闻：
- 地球信息科学
- 遥感
- 测绘地理信息
- 北斗与时空服务
- 实景三维
- 数字孪生
- 空天信息
- 广西本地产业政策、园区和项目

### 4.2 来源白名单原则
仅允许白名单来源进入自动发布链路，优先覆盖：
- 政府部门官网
- 行业协会官网
- 科研机构官网
- 高校官网
- 国际权威机构官网
- 行业重点企业官网

### 4.3 首版建议来源类型
- 广西壮族自治区自然资源厅
- 广西壮族自治区人民政府
- 中华人民共和国自然资源部
- 中国地理信息产业协会
- 中国测绘学会
- 中国科学院空天信息创新研究院
- 武汉大学测绘学院
- Open Geospatial Consortium
- European Space Agency
- 超图软件等重点企业官网

### 4.4 自动发布规则
新闻链路如下：
1. 白名单抓取
2. 清洗标准化
3. 去重
4. 自动分类
5. 关键词抽取
6. 详情页生成
7. 词云刷新
8. 知识图谱刷新

### 4.5 可信约束
每条发布新闻必须展示：
- 来源名
- 原文链接
- 发布时间
- 分类

首版要求：
- 白名单外来源不进入前台发布数据集
- 同标题同来源新闻不重复展示
- 新闻正文不做事实改写
- 摘要优先采用原文导语或规则清洗结果

## 5. 词云功能需求

### 5.1 目标
基于最近 30 天已发布新闻提取关键词，形成产业热点可视化视图。

### 5.2 展示方式
- 支持综合热词视图
- 支持企业、技术、政策三个栏目分别展示词云
- 点击词项后跳转到新闻列表页，并自动带入对应关键词筛选

### 5.3 数据规则
- 关键词来源于标题、摘要、规则词典和人工预置领域短语
- 对停用词进行过滤
- 对广西相关词条和高频领域词进行加权

## 6. 广西专题知识图谱需求

### 6.1 目标
构建聚焦广西产业专题的轻量级知识图谱，帮助用户理解区域政策、机构、园区、项目和技术之间的关系。

### 6.2 实体类型
首版固定以下实体类型：
- 政策
- 企业
- 机构
- 高校
- 园区
- 项目
- 技术
- 地区

### 6.3 关系生成逻辑
- 根据新闻中的实体共现关系生成边
- 广西本地实体优先保留在图谱中
- 每条边必须至少能追溯到一篇证据新闻
- 点击节点后展示实体简介、关联关系和证据新闻

### 6.4 广西专题范围
重点覆盖：
- 广西本地政策行动
- 自治区级机构
- 北部湾等重点区域
- 北斗产业园等载体
- 实景三维、遥感、数字孪生、北斗应用等核心技术

## 7. 视觉与交互要求

### 7.1 风格方向
- 参考研究机构门户风格与信息组织方式
- 不做参考网站的品牌复刻和像素级模仿
- 采用蓝绿色科技研究风格
- 信息密度较高，强调图文卡片和模块化布局

### 7.2 交互要求
- 提供顶部导航和语言切换按钮
- 中文和英文界面切换时保留相同页面结构
- 新闻内容按原文语言展示，不做全文机器翻译
- PC 与移动端自适应

### 7.3 内容页体验
- 详情页保留站内摘要与结构化信息
- 用户可以直接跳转原文来源
- 来源说明页展示白名单与日志信息

## 8. 非功能要求与验收标准

### 8.1 性能与稳定性
- 首页、新闻页、知识图谱页需支持静态预渲染
- 前台只读取结构化数据集，不直接依赖后台管理系统
- 定时更新任务执行后自动刷新新闻、词云、图谱和日志数据

### 8.2 兼容性
- 支持最新版本 Chrome 和 Edge
- 支持主流 iPhone 和 Android 浏览器
- 移动端不允许出现严重布局断裂

### 8.3 验收标准
#### 新闻采集验证
- 仅白名单来源可进入发布数据集
- 白名单外来源不会出现在前台
- 重复新闻不会重复展示

#### 可信性验证
- 每条新闻展示来源名、原文链接、发布时间和分类
- 企业、技术、政策分类结果可抽样检查

#### 交互验证
- 中英界面切换可用
- 新闻原文语言保持不变
- 列表筛选、排序、搜索、词云跳转均可用

#### 图谱验证
- 广西实体可按类型过滤
- 关系边可追溯证据新闻
- 点击节点可查看简介、关系和证据文章

#### 运行验证
- 默认每日 2 次执行更新任务
- 单次任务执行后可完成：新增抓取、去重、词云刷新、图谱刷新、日志输出

## 当前仓库实现说明

当前仓库已落地以下内容：
- Next.js + TypeScript 前台项目骨架
- 中英双语站点路由与页面结构
- 新闻列表、详情、词云、知识图谱、来源说明、关于页面
- 白名单来源配置、种子回退数据与实时抓取脚本
- 数据生成脚本：优先抓取实时新闻，失败时自动回退种子数据，并输出新闻、词云、图谱、来源和日志 JSON
- 可通过 `npm run data:refresh` 刷新结构化数据
- 可通过 `npm run build` 完成生产构建验证
