# 系统架构文档

## 系统概述

广西地信新闻系统是一个全栈新闻聚合平台，专注于地球信息科学、遥感、测绘、GIS等领域的新闻采集、处理和展示。

## 技术栈

### 前端
- **框架**: Next.js 15 + React 19
- **语言**: TypeScript
- **样式**: Tailwind CSS
- **地图**: 百度地图API

### 后端
- **语言**: Java 21
- **框架**: Spring Boot 3
- **构建工具**: Maven
- **ORM**: MyBatis-Plus
- **数据库**: MySQL 8
- **迁移工具**: Flyway

### 基础设施
- **部署**: 前后端分离部署
- **端口**: 前端3000，后端8080
- **数据库**: MySQL 3306

## 核心流程

### 1. 新闻抓取流程

```
启动触发 (自动/手动)
    ↓
CrawlWindowService.resolve()
    ├─ 首次运行: 从配置的起始时间开始
    └─ 增量运行: 上次结束时间 - 10分钟
    ↓
SourceBootstrapService.ensureSources()
    └─ 从 datasets/config/sources.json 加载 active 且绑定实体白名单的来源
    ↓
并行处理 (线程池, 默认20个线程)
    ↓
对每个来源:
    ├─ SourceCrawlerService.crawl()
    │   ├─ 支持 HTML、RSS/Atom、API、sitemap
    │   ├─ 列表选链先执行详情页 URL 判断，过滤栏目页、about、support 等非新闻页
    │   ├─ 只有能解析发布时间且落在窗口内的详情页才输出候选，避免 Java 抓取继续留下 `missing_published_at`
    │   └─ 日志记录“发现链接/条目”和日期缺失、窗口外、非详情页等过滤原因，发现链接数不等于入库数
    ├─ NewsCleaningService.clean()
    │   ├─ 清理标题、正文和摘要
    │   ├─ 从正文段落中选择天然包含硬词的摘要
    │   ├─ 提取关键词
    │   ├─ 识别地区标签
    │   ├─ 分类 (enterprise/technology/policy)
    │   └─ 生成内容哈希
    ├─ NewsReviewService.review()
    │   ├─ 验证来源启用且绑定实体白名单
    │   ├─ 验证日期范围
    │   ├─ 验证URL是详情页且可访问 (带重试)
    │   ├─ 严格正文相关性检查
    │   ├─ 验证摘要包含硬词且来自正文
    │   └─ 返回接受/拒绝决定
    ├─ NewsPersistenceService.recordCandidate()
    │   └─ 记录所有候选到 news_candidates 表
    └─ NewsPersistenceService.persistAccepted()
        ├─ 检查重复 (sourceId + canonicalUrl)
        ├─ 检查版本重复 (contentHash)
        └─ 插入新版本或标记为重复
    ↓
汇总统计
    ├─ 候选数、接受数、拒绝数
    ├─ 插入版本数、重复版本数
    └─ 失败来源数、部分成功来源数
    ↓
更新 crawl_runs 表
    ↓
刷新派生数据集快照 (如果有新数据)
```

### 2. 新闻查询流程

```
用户请求 → Next.js SSR
    ↓
lib/backend-data.ts
    ├─ 尝试调用后端API
    └─ 失败则回退到本地JSON
    ↓
后端 NewsController.list()
    ↓
NewsQueryService.list()
    ├─ 应用过滤条件
    │   ├─ 日期范围 (>= minimumPublishedAt)
    │   ├─ 分类、来源、地区
    │   ├─ 全文搜索
    │   └─ 广西相关过滤
    ├─ 排序 (最新/最旧)
    └─ 限制数量 (最多2000)
    ↓
NewsMapper.selectCurrentArticleRows()
    └─ 复杂SQL连接 news + news_versions + sources
    ↓
转换为 ArticleDto
    ├─ 解析JSON字段 (keywords, regionTags, entityIds)
    └─ 格式化日期
    ↓
返回JSON响应
    ↓
前端渲染
```

## 关键设计决策

### 1. 并行抓取

**决策**: 使用线程池并行处理200个来源

**原因**:
- 串行处理耗时过长（每个来源5-10秒，总计16-33分钟）
- 大部分时间花在网络I/O上

**实现**:
- 使用 `ExecutorService` 创建固定大小线程池
- 默认并行度20，可通过配置调整
- 使用 `ConcurrentHashMap` 和 `AtomicInteger` 保证线程安全

**权衡**:
- 优点: 显著提升性能（预计5-10倍加速）
- 缺点: 增加数据库连接压力，可能被目标网站限流
- 缓解: 配置合理的并行度，数据库连接池足够大

### 2. URL验证重试机制

**决策**: 添加指数退避重试（最多3次）

**原因**:
- 网络不稳定可能导致临时失败
- 某些服务器可能有短暂的限流

**实现**:
- 初始退避1秒，每次重试翻倍（1s, 2s, 4s）
- 只对IOException重试，RuntimeException直接失败

### 3. 白名单准入与严格审核

**决策**: 来源准入、正文相关性、摘要来自正文、摘要硬词、日期范围和 URL 可访问性都是硬门槛。

**实现**:
- `SourceCatalogService` 从 `sources.json` 同步来源时解析 `whitelistEntityId`，无法绑定 `entity-whitelist.json`、类型不允许或命中新华网/人民网/央视网/科技日报/中国新闻网/中国政府网等综合媒体门户黑名单的来源会被停用。
- `NewsReviewService` 不再读取 `requireKeywordMatch:false` 作为放行条件。
- `StrictRelevanceChecker` 使用正文、核心主题词和实体证据判断强相关。
- `NewsComplianceService` 为质量审计和全规则清理复用同一套来源、来源域名、日期、URL、正文、摘要硬词和相关性判断。
- 入库版本写入 `url_status=accessible`、`url_verified_at`、`final_url` 和 `body_text`。
- 新闻页使用 `/api/v1/news/page` 分页读取，前端后端数据请求使用 5 分钟再验证缓存；搜索结果只高亮文本节点，不注入 HTML。

**验收**:
- `NewsQualityAuditService` 输出唯一合格新闻数、白名单覆盖率、正文缺失、摘要非正文、正文强相关失败、中文/广西数量、URL 可访问率和拒绝原因。
- `CrawlBackfillService` 可按月/季度回溯抓取，但只从允许的领域政府、高校、科研、协会、企业等来源补量，按广西、国内优先。

### 4. 抓取窗口重叠

**决策**: 每次抓取从"上次结束时间 - 10分钟"开始

**原因**:
- 防止因时间同步问题遗漏新闻
- 某些网站发布时间可能不精确

**实现**:
- `CrawlWindowService` 查询最近成功的抓取运行
- 减去配置的重叠时间（默认10分钟）
- 若历史 `window_end_at` 晚于当前批次结束时间，则夹紧为当前结束时间前的重叠窗口，避免反向窗口导致所有候选被日期过滤

### 5. 内容去重策略

**决策**: 两级去重

**原因**:
- 同一新闻可能被多次抓取
- 新闻内容可能更新

**实现**:
1. **新闻级去重**: `(sourceId, canonicalUrl)` 唯一约束
2. **版本级去重**: `(newsId, contentHash)` 唯一约束

**效果**:
- 同一URL的新闻只有一条记录
- 内容更新时创建新版本
- 完全相同的内容不重复存储

### 6. 广西相关判断

**决策**: 基于地区标签精确匹配

**原因**:
- 原逻辑"任何非全国"会误判其他省份
- 需要精确识别广西相关新闻

**实现**:
- 定义广西地区术语列表（广西、南宁、柳州等）
- 检查 `regionTags` 是否包含这些术语

## 性能优化

### 已实现
1. **并行抓取**: 20个线程并行处理来源
2. **URL验证重试**: 减少因临时失败导致的误判
3. **数据库索引**: 关键字段建立索引

### 待优化
1. **JSON解析缓存**: 避免每次审核都解析来源的whitelist
2. **字符串处理优化**: 预编译正则表达式，减少重复拼接
3. **连接池调优**: 根据并行度调整数据库连接池大小

## 安全考虑

### 认证
- 内部API使用 `X-Internal-Token` 头认证
- 令牌通过环境变量配置

### 数据验证
- 所有外部输入都经过验证
- URL验证防止SSRF攻击
- SQL使用参数化查询防止注入

### 限流
- 抓取并行度可配置，防止过载
- URL验证有超时限制（12秒）

## 监控和日志

### 抓取日志
- 每次抓取运行记录到 `crawl_runs` 表
- 每个来源的详细统计记录到 `crawl_run_sources` 表
- 包含候选数、接受数、拒绝数、错误信息

### 候选日志
- 所有抓取的候选记录到 `news_candidates` 表
- 包含原始数据、清洗后数据、审核结果

### 应用日志
- Spring Boot标准日志
- 错误和异常自动记录

## 配置管理

### 环境变量
- `.env.local`: 本地开发配置
- `.env`: 默认配置
- 生产环境通过系统环境变量注入

### 配置文件
- `application.yml`: Spring Boot配置
- `datasets/config/sources.json`: 新闻来源配置
- `datasets/config/entity-whitelist.json`: 实体白名单
- `datasets/seed/graph-research-base.mjs`: 知识图谱研究底库，生成 `广西总览 + 14 个设区市`，每个城市至少 `80` 个具体节点，并覆盖 `主体、目标、内容、活动、评价` 五类要素。

### 知识图谱派生数据
- 图谱生成由 `scripts/update-datasets.mjs` 合并研究底库、新闻证据和研究证据，输出 `datasets/generated/knowledge-graph.json`。
- 城市图谱节点必须具体到政府部门、自然资源局、发改/工信/交通/应急/农业农村等部门、具体园区平台、高校科研机构、产业目标、卫星遥感/北斗时空基准/实景三维/测绘资质数据、三维建模/数据建库/AI 遥感解译等活动，以及结构优化/技术承载/公共服务效能等评价。
- `scripts/check-project.mjs` 对广西总览只含14市、每市83个节点、五类要素固定数量、城市直接关系、来源证据、禁止泛化词和桂林样例节点做硬性审计。
- 新闻来源配置要求保留广西 14 个市自然资源局；广西壮族自治区统计局作为统计公报、产业规模和市场主体结构来源；广西壮族自治区自然资源厅补充自然资源公报、测绘资质公告和复审换证入口。
- 前端图谱页支持 `region` 和 `class` 查询参数，广西总览只展示14个设区市，城市页按五类要素展示全量节点；`class` 取 `subject`、`goal`、`content`、`activity`、`evaluation` 时固定进入对应要素筛选，用于桌面截图验收和外部链接。
- `components/graph-explorer.tsx` 在客户端从生成数据派生城市图形图谱，不改变后端接口和 `datasets/generated/knowledge-graph.json` schema。图形视图使用本地依赖 `echarts@^5.5.1` 动态加载，并以 ECharts `graph` 的 SVG renderer 渲染。
- 城市图形图谱的展示结构为 `城市中心节点 + 5 个要素 Hub + 具体实体节点`，边关系为 `城市中心 -> 要素 Hub -> 具体实体节点`。5 个要素 Hub 只存在于前端视图层；文字图谱继续直接使用生成数据中的真实节点。
- 图形视图保留拖拽节点、悬浮完整名称 tooltip、点击详情联动和右侧五类要素多选过滤；筛选后只保留城市中心、选中 Hub、选中类别实体及对应边。
- Edge headless 验收脚本 `target/kg-edge-screenshots/edge-validate.cjs` 检查 ECharts SVG 容器、可见节点/边、tooltip、拖拽、筛选、无关系网络图入口、无控制台错误和文字图谱切换，截图输出到 `target/kg-edge-screenshots`。

## 故障恢复

### 抓取失败
- 部分来源失败不影响整体
- 下次抓取会重试失败的时间窗口

### 数据库故障
- 事务保证数据一致性
- 失败的抓取可以重新运行

### 前端降级
- 后端不可用时自动回退到本地JSON
- 用户体验不中断
